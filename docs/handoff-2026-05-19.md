# Handoff 文档 - 2026/05/19

## 主题：设施拖放功能实现

### 需求
让用户可以从右上角的设施列表中拖动图片到地图上，松开后在地图上以标记形式显示该设施图片。

### 实现概要

#### 修改文件
- `src/App.vue`

#### 主要改动

1. **新增常量**
   - `DEFAULT_MARKER_SIZE = 32` - 标记默认尺寸

2. **数据结构扩展**
   - `facilityList` 支持可选的 `width`、`height` 字段
   - 示例：`{ imageUrl: '/favicon.svg', description: '设施名称', width: 48, height: 48 }`
   - 默认不配置时自动使用 32x32

3. **新增拖放处理函数**
   - `handleFacilityDragStart(event, facility)` - 拖动开始时存储设施数据
   - `handleMapDragOver(event)` - 允许在地图上放置
   - `createFacilityMarker(facility, lngLat)` - 创建标记
   - `handleMapDrop(event)` - 放置时创建标记并添加到地图

4. **标记管理**
   - `placedMarkers` 数组存储所有放置的标记
   - 在 `onUnmounted` 中清理

5. **Template 修改**
   - 设施图片添加 `draggable="true"`
   - 添加 `@dragstart="handleFacilityDragStart($event, item)"`
   - 文字部分不加 draggable，实现"只在图片上能拖动"

6. **生命周期更新**
   - `onMounted`：添加地图容器的 `dragover` 和 `drop` 事件监听
   - `onUnmounted`：移除事件监听，清理 `placedMarkers`

### 使用方法

1. 鼠标左键按住设施列表中的图片（文字上不行）
2. 拖动到地图任意位置
3. 松开鼠标，标记会出现在该位置

### 验证方式

```bash
pnpm dev
```

打开浏览器测试：
- 从文字上拖动应该无效
- 从图片上拖动应该出现拖动效果
- 在地图上松开应该出现设施标记
- 标记位置应准确

### 注意事项

- 使用高德 `AMap.Marker` + `content` 选项自定义标记内容
- 坐标转换使用 `mapInstance.containerToLngLat()`
- 标记锚点设为 `center` 配合 `offset` 确保位置准确
- 图片添加 `pointer-events:none` 和 `draggable="false"` 防止标记内部的干扰

## 主题：禁用浏览器默认拖拽预览效果

### 问题描述
设施列表中的拖拽功能会同时显示自定义预览和浏览器默认的半透明拖拽预览效果，需要关闭浏览器自带的预览。

### 解决方案
将 HTML5 Drag and Drop API 改为鼠标事件（mousedown/mousemove/mouseup）实现，设置 `draggable="false"` 彻底禁用浏览器原生拖拽行为。

### 具体修改内容

#### 1. 模板部分（设施图片元素）
- `draggable="true"` → `draggable="false"`
- `@dragstart` → `@mousedown`
- 事件处理函数：`handleFacilityDragStart` → `handleFacilityMouseDown`

#### 2. 脚本部分
**删除的函数：**
- `handleFacilityDragStart` - 原 HTML5 拖拽开始处理
- `handleMapDragOver` - 原 HTML5 拖拽经过地图处理
- `handleMapDrop` - 原 HTML5 放置处理
- `handleDocumentDragOver` - 原文档拖拽移动处理
- `cleanupDrag` - 原清理函数

**新增的函数：**
- `handleFacilityMouseDown(event, facility)` - 鼠标按下开始拖拽
- `handleDocumentMouseMove(event)` - 鼠标移动更新预览位置
- `handleDocumentMouseUp(event)` - 鼠标释放判断放置
- `cleanupMouseDrag()` - 清理鼠标拖拽状态

**修改的生命周期：**
- `onMounted`: 移除了 `dragover` 和 `drop` 事件监听器的添加
- `onUnmounted`: 将 `cleanupDrag()` 改为 `cleanupMouseDrag()`，移除了事件监听器的移除

#### 3. CSS 部分
在 `.facility-image` 样式中添加了：
- `user-select: none;`
- `-webkit-user-select: none;`

防止图片被意外选中。

### 预览图片位置处理
保留了原有的预览位置计算逻辑：
```javascript
left: ${clientX - width / 2}px
top: ${clientY - height / 2}px
```
确保预览图片的中心始终位于鼠标光标位置。

### 放置逻辑变更
- **原逻辑**：通过 HTML5 `drop` 事件触发，从 `dataTransfer` 获取数据
- **新逻辑**：通过 `mouseup` 事件触发，判断鼠标位置是否在地图容器内，直接使用 `draggedFacility.value` 获取数据

### 验证结果
- ✅ 构建无错误
- ✅ 拖拽时只显示自定义预览元素
- ✅ 预览图片中心跟随鼠标光标
- ✅ 在地图内释放可正确放置标记
- ✅ 在地图外释放不放置标记

## 主题：修复设施拖放创建标记失败（TypeError: Cannot read properties of null）

### 问题描述
从设施列表拖拽设施到地图时无法创建标记，控制台报错：
```
放置设施失败: TypeError: Cannot read properties of null (reading 'width')
    at createFacilityMarker (App.vue:461:26)
    at HTMLDocument.handleDocumentMouseUp (App.vue:449:20)
```

### 根本原因
在 `handleDocumentMouseUp` 函数中，`cleanupMouseDrag()` 在第430行被提前调用，这会立即将 `draggedFacility.value` 设置为 `null`。后续代码（第446行）尝试获取 `draggedFacility.value` 时已为 null，导致 `createFacilityMarker` 函数中访问 `facility.width` 时报错。

### 修复内容
**文件**: `src/App.vue`
**函数**: `handleDocumentMouseUp`

将 `cleanupMouseDrag()` 的调用移至函数末尾，使用 `finally` 确保即使发生异常也会清理状态：

1. 在 `!mapInstance` 提前返回前添加清理
2. 在地图范围外提前返回前添加清理
3. 将主要清理逻辑移到 `finally` 块中

### 代码变更
```javascript
// 修复前
cleanupMouseDrag();  // ← 提前清理了
if (!mapInstance) return;
// ... 使用 draggedFacility.value 的代码 ...

// 修复后
if (!mapInstance) {
  cleanupMouseDrag();
  return;
}
// ... 使用 draggedFacility.value 的代码 ...
finally {
  cleanupMouseDrag();
}
```

### 验证
- ✅ `pnpm build` 构建成功
- ✅ 设施可正常拖拽放置到地图
